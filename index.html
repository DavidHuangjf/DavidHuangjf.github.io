bluetooth_on_two_Desktop_OS

发表于 2017-07-06

多系统使用同一蓝牙设备

由于蓝牙是一对一传输，当系统连接蓝牙设备的时候，会随机生成一个link ID，因每次配对都会改变其link ID，所以导致不同OS的蓝牙不能共用，即使是同一蓝牙芯片也不可以。

步骤一

在操作系统下建立的蓝牙配对。【必须先配对】

步骤二

OSX

sudo defaults read /private/var/root/Library/Preferences/blued.plist

​ 找到 link ID 的 32位的16进制数字 ,将其Little Endian to Big Endian：

​ 例：18E8146974DD86EF576669A58FD5ECF2 > F2ECD58FA5696657EF86DD746914E818

Linux

​ 找到 link ID key 的键值， 32位的16进制数字

步骤三

Windows系统中使用微软官网下载PSTools

以管理员身份运行命令行工具，在PSTools解压缩目录下执行打开注册表

PsExec.exe -s -i regedit

在注册表中找到

HKEY_LOCAL_MACHINE\SYSTEM\CurrentControlSet\services\BTHPORT\Parameters\Keys\

下面的蓝牙设备ID名称并更改其键值。

步骤四

重启电脑，蓝牙设备不用再配对就能用啦！ 可以不用restart

OpenSSL_Update_Shadowsocks_Error

发表于 2017-07-06

Kali 2.0 update后 Shadowsocks 服务报错

报错信息

AttributeError: /usr/lib/x86_64-Linux-gnu/libcrypto.so.1.1: undefined symbol: EVP_CIPHER_CTX_cleanup

经查询，这个问题是由于openssl新版本废弃了EVP_CIPHER_CTX_cleanup函数引起的

修改方法：

  1. locate openssl.py
  2. 打开文件：/usr/local/lib/python2.7/dist-packages/shadowsocks/crypto/openssl.py
  3. 搜索cleanup
  4. 将第52行libcrypto.EVP_CIPHER_CTX_cleanup.argtypes = (c_void_p,)

​ 改为libcrypto.EVP_CIPHER_CTX_reset.argtypes = (c_void_p,)

  1. 再次搜索cleanup位于111行，将libcrypto.EVP_CIPHER_CTX_cleanup(self._ctx)

​ 改为libcrypto.EVP_CIPHER_CTX_reset(self._ctx)

  1. 保存并退出
  2. 启动shadowsocks服务： sslocal -c ss.config​
